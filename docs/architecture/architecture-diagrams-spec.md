## 图1: 总体架构分层图

### 描述
展示平台从客户端到基础设施的完整分层架构，共8层。

### 层次结构（从上到下）

1. **客户端层**
   - 包含5个组件: 小程序、App、PC管理台、POS终端、开放API
   - 横向排列，用不同图标表示不同终端类型

2. **接入层**
   - 微前端容器(Module Federation)
   - 子组件: 统一门户、动态加载、版本管理
   - 用虚线框表示前端聚合层

3. **API网关层**
   - 用Go语言标签标注
   - 包含5个功能模块: 路由、限流、鉴权、灰度、监控
   - 用横向方框排列

4. **Istio Service Mesh控制面**
   - 包含6个组件: Pilot、Citadel、Galley、Kiali、Prometheus、Jaeger
   - 用网格状排列
   - 标注"控制面"标签

5. **三大中台（并行展示）**

   **左列 - 业务中台(Java)**:
   - 10个领域服务纵向排列
   - 商品域、订单域、库存域、会员域、营销域
   - 供应链、财务、物流、生产、售后

   **中列 - 技术中台(Go)**:
   - 10个基础设施服务纵向排列
   - 统一网关、消息中心、任务调度、配置中心
   - 分布式ID、缓存服务、文件服务、搜索服务
   - 多租户管理、可观测平台

   **右列 - 数据中台**:
   - 7个数据服务纵向排列
   - 实时计算(Flink)、批处理(Spark)、数据湖(Hudi)
   - 特征平台、AI平台(TF)、推荐引擎、BI分析引擎

6. **基础设施层(K8s多集群)**

   **上半部分**:
   - 控制面集群（管理/运维）
   - 数据面集群（业务运行）
   - 用虚线分隔两种集群

   **下半部分**:
   - 基础设施组件横向排列
   - PostgreSQL、Redis、Kafka、ElasticSearch、MinIO等
   - 用图标+文字标注

### 连接线说明
- 客户端层→接入层: 粗实线，表示主要流量入口
- 接入层→API网关层: 粗实线
- API网关层→Istio控制面: 实线
- Istio控制面→三大中台: 分叉实线，表示流量分发
- 三大中台→基础设施层: 实线，表示数据存储

### 颜色建议
- 客户端层: 蓝色系（前端色调）
- 接入层: 浅蓝色
- API网关层: 深绿色（Go语言色调）

