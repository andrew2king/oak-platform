## 一、项目概述

### 1.1 项目定位

Oak Platform（新零售平台）是面向超大型品牌连锁零售企业的分布式电商平台，具备商业化输出能力。

### 1.2 核心能力

| 能力模块 | 描述 |
|---------|------|
| **业务中台** | 全链路业务能力：交易、营销、供应链、财务、生产、物流、售后 |
| **技术中台** | 商业化输出的基础设施，支持多租户架构 |
| **数据中台** | 全链路数据智能：实时计算 + AI/ML + 推荐引擎 |
| **全渠道营销** | 线上线下会员、库存、价格、促销完全打通 |
| **O2O 融合** | 到店 + 到家 + 即时零售 + 社交电商（直播、社群、分销） |

### 1.3 业务规模

- 门店规模：1000+ 家门店
- 运营范围：全国布局
- 架构要求：高可用、分布式、多活部署

## 二、技术选型

### 2.1 技术栈决策

| 层级 | 技术栈 | 说明 |
|-----|--------|------|
| **业务中台** | Java (Spring Cloud) | 业务复杂度高，Java 生态成熟 |
| **技术中台** | Go | 高性能基础设施，资源消耗低 |
| **数据中台** | Flink + Spark + TensorFlow | 实时计算 + 批处理 + AI |
| **Service Mesh** | Istio | 全链路治理，可观测性强 |
| **容器编排** | Kubernetes | 多集群部署，支持多活 |

### 2.2 多租户模式

采用**独立部署模式**：
- 大客户：独立集群部署，完全物理隔离
- 中小客户：共享资源池，逻辑隔离
- 支持客户自选隔离级别

### 3.2 核心设计原则

| 原则 | 说明 |
|------|------|
| **领域驱动设计 (DDD)** | 业务中台按领域划分，每个域独立演进 |
| **多租户隔离** | 大客户独立集群，中小客户共享资源池 |
| **可观测优先** | Istio + Prometheus + Jaeger 全链路追踪 |
| **事件驱动** | 核心业务流程采用事件驱动，支持最终一致性 |
| **API First** | 所有服务通过 API 网关暴露，支持开放平台 |

### 7.3 领域依赖关系详解

#### 核心域（商品域 + 订单域）

这两个领域是整个业务中台的基石，几乎所有其他领域都依赖它们：

- **商品域**：提供商品基础信息、价格策略、SKU 结构
- **订单域**：记录交易行为，驱动上下游业务流转

#### 第一层依赖域（库存域 + 会员域）

- **库存域**：
  - 依赖商品域：获取商品 SKU 信息
  - 依赖订单域：订单扣减库存、退货回补库存

- **会员域**：
  - 依赖订单域：消费记录影响会员等级、积分累积

#### 第二层依赖域（营销域 + 供应链域）

- **营销域**：
  - 依赖商品域：活动商品范围、促销定价
  - 依赖会员域：会员专属活动、积分兑换
  - 依赖订单域：订单优惠计算、活动效果统计

- **供应链域**：
  - 依赖商品域：采购商品清单
  - 依赖库存域：库存预警触发采购、入库更新库存

#### 第三层依赖域（财务域 + 物流域）

- **财务域**：
  - 依赖订单域：订单金额结算、退款处理
  - 依赖供应链域：采购成本核算

- **物流域**：
  - 依赖订单域：配送任务生成
  - 依赖库存域：发货扣减、门店自提库存预留

#### 外围域（生产域 + 售后域）

- **生产域**：
  - 依赖供应链域：原材料采购、生产计划排产

- **售后域**：
  - 依赖订单域：售后订单关联、退款处理
  - 依赖会员域：会员售后记录、满意度追踪

### 10.2 O2O核心场景矩阵

| **即时零售** | 线上下单→即时达 | 第三方骑手配送 | 骑手调度、时效保证 |
| **直播带货** | 直播间下单→履约 | 多渠道履约路由 | 直播间数据、库存联动 |

### 7.2 关键技术决策记录

| 决策点 | 选择 | 理由 |
|-------|-----|------|
| 架构模式 | 微前端 + Service Mesh | 现代化、可观测性强、多语言支持 |
| 业务中台语言 | Java | 生态成熟、适合复杂业务 |
