## Slide 2: 项目背景

### 市场趋势
- 新零售：线上线下融合已成趋势
- 数字化：传统零售企业数字化转型需求强烈
- 平台化：企业需要自建平台掌控数据和用户

### 项目定位
- 面向超大型品牌连锁零售企业
- 1000+门店规模，全国布局
- 具备商业化输出能力，服务外部客户

## Slide 3: 核心能力概览

| 能力 | 描述 |
|-----|------|
| 业务中台 | 全链路：交易 + 营销 + 供应链 + 财务 + 生产 + 物流 + 售后 |
| 技术中台 | 商业化输出，多租户架构，高性能基础设施 |
| 数据中台 | 实时计算 + AI/ML + 推荐引擎 + BI分析 |
| 全渠道营销 | 线上线下会员、库存、价格、促销完全打通 |
| O2O融合 | 到店 + 到家 + 即时零售 + 社交电商 |

---

## Slide 4: 技术选型决策

### 技术栈
- **业务中台**: Java (Spring Cloud) - 生态成熟，复杂业务适配
- **技术中台**: Go - 高性能，低资源消耗
- **数据中台**: Flink + Spark + TensorFlow
- **Service Mesh**: Istio - 全链路治理
- **容器编排**: Kubernetes - 多集群部署

### 多租户模式
- 大客户：独立集群部署
- 中小客户：共享资源池
- 客户可自选隔离级别

## Slide 6: Service Mesh架构

### Istio核心组件
- **Pilot**: 流量管理、服务发现
- **Citadel**: 安全认证、证书管理
- **Galley**: 配置验证、分发
- **Kiali**: 服务拓扑可视化
- **Prometheus**: 指标采集
- **Jaeger**: 分布式追踪

### 优势
- 多语言无缝集成（Java + Go）
- 全链路可观测性
- 统一流量治理
- 故障注入测试

## Slide 8: 业务中台领域依赖关系图

- 物流域 ← 订单域、库存域

### 外围域
- 生产域 ← 供应链域
- 售后域 ← 订单域、会员域

## Slide 10: 技术中台核心服务详解

| 服务 | 技术选型 | 核心功能 | 商业化能力 |
|-----|---------|---------|-----------|
| 统一网关 | Kong (Go) | API路由、协议转换 | 租户级路由策略 |
| 消息中心 | Kafka | Topic管理、消息订阅 | 租户独立Topic |
| 任务调度 | Temporal | 分布式任务、失败重试 | 任务租户隔离 |
| 配置中心 | Nacos | 动态配置、热更新 | 租户级配置空间 |
| 分布式ID | Snowflake | 全局唯一ID | 100万+/秒并发 |
| 缓存服务 | Redis Cluster | 缓存、分布式锁 | 租户配额管理 |
| 文件服务 | MinIO | 文件上传/下载、CDN | 租户独立存储 |
| 搜索服务 | ES | 索引管理、搜索DSL | 租户独立索引 |
| 多租户管理 | Go服务 | 租户创建、资源分配 | 核心商业化 |
| 计费系统 | Go服务 | 用量统计、账单生成 | 商业化计费 |
| 可观测平台 | Prometheus+EFK | 日志/指标/链路 | 租户数据隔离 |

## Slide 15: 多集群部署架构

### 数据处理链路
```
数据采集 → 实时计算 → 数据湖 → AI平台 → 应用服务
    ↓         ↓         ↓         ↓
```

## Slide 13: 核心设计原则

| 原则 | 说明 |
|-----|------|
| 领域驱动设计(DDD) | 业务中台按领域划分，独立演进 |
| 多租户隔离 | 大客户独立集群，中小客户共享 |
| 可观测优先 | Istio + Prometheus + Jaeger |
| 事件驱动 | 核心流程采用事件驱动 |
| API First | 所有服务通过API网关暴露 |
